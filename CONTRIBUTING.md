# Участие в разработке КиберэдэН

> Этот документ описывает правила и процесс участия в проекте.

---

## Кто может участвовать

Проект закрытый на этапе pre-production. Участие только по приглашению nazrOS.

---

## Структура веток

main          — стабильная версия, только через PR
dev           — основная ветка разработки
feature/*     — новые механики (feature/quest-system)
fix/*         — исправления (fix/character-needs)
docs/*        — документация (docs/scenario-karma)

---

## Правила коммитов

Формат:

тип(область): описание на русском

Типы:
- feat — новая механика или функция
- fix — исправление бага
- docs — документация
- art — ассеты, концепт-арт
- refactor — рефакторинг без новой логики
- wip — работа в процессе

Примеры:

feat(quests): добавлена механика квеста «помой посуду»
fix(character): исправлен таймер убывания потребности сон
docs(gdd): обновлён GDD до v0.4
art(concept): добавлен концепт-арт комнаты Кибера

---

## Процесс работы

1. Создать ветку от dev
2. Сделать изменения
3. Commit по формату выше
4. Pull Request в dev
5. Ревью от nazrOS
6. Merge

---

## Что нельзя

- Пушить напрямую в main
- Коммитить бинарные ассеты без Git LFS
- Менять GDD без согласования
